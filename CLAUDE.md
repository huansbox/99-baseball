# 隕石數學防衛隊 Meteor Math Defense

## 專案概要

- **類型**：9x9 乘法練習遊戲（太空隕石主題）
- **平台**：iPad Safari 觸控優化
- **部署**：GitHub Pages 靜態網站
- **技術**：單檔 HTML + CSS + Vanilla JS（無框架依賴）

## 遊戲機制

### 三關卡制

三回合皆 10 題，僅間隔與落速遞增。原始 15/20 題設定保留於程式碼註解中。

| 關卡 | 題數 | 基礎間隔 | 落速 (px/frame) | 倍率 |
|------|------|----------|-----------------|------|
| 1    | 10   | 4s       | 0.6 ~ 0.9       | x1.0 |
| 2    | 10   | 3s       | 0.9 ~ 1.2       | x1.5 |
| 3    | 10   | 2s       | 1.2 ~ 1.5       | x2.0 |

### 動態生成機制

根據畫面上未答隕石數量自動調整生成間隔與落速，避免堆積過多。

**間隔公式**：未答 ≥ 3 顆時，每多 1 顆 +1 秒
```
超額 = max(0, 未答數 - 2)
實際間隔 = 基礎間隔 + 超額 × 1s
```

**落速公式**：未答 ≥ 3 顆時，落速減緩
```
落速 = speedMin × 0.9^超額（固定值，非隨機）
```

| 未答隕石數 | 額外間隔 | 落速變化 |
|-----------|---------|---------|
| 0-2       | +0s     | 正常隨機 |
| 3         | +1s     | ×0.9    |
| 4         | +2s     | ×0.81   |
| 5         | +3s     | ×0.73   |

### 題庫分類（同 99times）

- **Easy**：n×2 所有組合 + 3×3（16 題）
- **Hard**：其餘 3~9 互乘組合（48 題）

### 智慧出題配額表

三回合皆 10 題，配額依據「錯題庫實際數量」動態查表。原始 15/20 題配額保留於程式碼註解中。

| 錯題數 | 錯題 | easy | hard | 總計 |
|--------|------|------|------|------|
| 0      | 0    | 2    | 8    | 10   |
| 1      | 1    | 2    | 7    | 10   |
| ≥2     | 2    | 1    | 7    | 10   |

### 得分區域

| 區域 | Y 範圍 | 基礎分 | 關卡倍率 |
|------|--------|--------|---------|
| 天空 | 0-30%  | 100    | ×1.0/1.5/2.0 |
| 高空 | 30-60% | 80     | ×1.0/1.5/2.0 |
| 低空 | 60-85% | 60     | ×1.0/1.5/2.0 |
| 地面 | 85%+   | 40     | **不乘倍率** |

### Combo 計分系統

**即時加成**：連續答對時，每題獲得額外加成
```
倍率 = min(2.0, 1 + (combo - 1) × 0.1)
```
| 連續 | 倍率 |
|------|------|
| 1    | ×1.0 |
| 2    | ×1.1 |
| 3    | ×1.2 |
| ...  | ...  |
| 10   | ×2.0 |

**連續獎勵**：回合結束時，連對 5+ 題額外獎勵
```
獎勵 = 連續題分數總和 × (1 + (combo - 5) × 0.2)
```
| 連對 | 獎勵倍率 |
|------|---------|
| 5    | ×1.0    |
| 6    | ×1.2    |
| ...  | ...     |
| 10   | ×2.0    |

**全對獎勵**：遊戲結束時
- 2 回合全對：該 2 回合分數 ×1 額外加
- 3 回合全對：總分 ×1 額外加（總分翻倍）

**STREAK 顯示**：連續 2+ 題時畫面中央顯示，此時隕石避開中央出現

## 核心類別

### StatsManager
- `getStats()` → 取得各題答對次數
- `getErrors()` → 取得錯題庫
- `addError(key)` → 加入錯題庫（streak=0）
- `incrementCorrect(key)` → 答對次數 +1
- `incrementStreak(key)` → 連續答對 +1，≥3 移出錯題庫
- `selectFromPool(pool, count, excludeKeys)` → 智慧選題

### RecordsManager
- `getRecords()` → 取得前 10 名記錄
- `saveRecord(score)` → 儲存並回傳排名
- `getHighScore()` → 取得最高分
- `deleteRecord(id)` → 刪除指定記錄

### Game
- `prepareStageQuestions()` → 依配額表生成回合題目
- `generateQuestion()` → 從預生成列表取題
- `handleGrounded(meteor)` → 落地加入錯題庫、中斷連續
- `handleMiss()` → 單隕石答錯偵測、中斷連續
- `destroyMeteor(meteor)` → 計算 combo 分數、更新統計
- `breakCombo()` → 結算連續獎勵並重置
- `showCombo()` / `hideCombo()` → STREAK 顯示控制
- `showStageResult()` → 回合結算畫面
- `showStats(tab, multiplier)` → 統計頁面

## 錯題追蹤機制

### 觸發條件
1. **隕石落地**：該題加入錯題庫
2. **單隕石答錯**：畫面只有 1 顆時答錯，加入錯題庫

### 移除條件
- 連續答對 3 次（streak ≥ 3）

### 資料結構
```javascript
// localStorage 持久化
questionStats: { "2×3": 5, "7×8": 1, ... }
errorBank: [{ key: "7×8", streak: 0 }, ...]
```

## 技術決策

| 項目 | 選擇 | 理由 |
|------|------|------|
| 架構 | 單檔 HTML | 簡化部署、易於維護 |
| 動畫 | CSS + DOM | 簡單效果足夠、無需 Canvas |
| 存儲 | localStorage | 離線可用、無後端 |
| 迴圈 | requestAnimationFrame | 流暢 60fps |
